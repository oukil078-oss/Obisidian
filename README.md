## Quick Start Guide 🚀

### When I Clone a New GitHub Project:

1. Create a project note using the `New Project.md` template
2. Link it to your [[Profile]] note in the "Active Projects" table
3. Add relevant tags (`#github`, `#python`, `#webapp`, etc.)
4. The Obsidian graph will automatically show connections!

### Graph Customization 💎

In `.obsidian/community-plugins.json`, enable these plugins:

- **Templater** - Use the templates I created
- **Dataview** - Query your notes and create dynamic tables
- **Graph View Enhancements** - Make the graph prettier with colors and layouts
- **Quick Add** - Create notes from hotkey (optional, nice for organization)

### Recommended Graph Settings

For a beautiful, organized graph:

- **Node Colors:** Color by tag type (`#project` = blue, `#tool` = green, `#resource` = purple)
- **Arrow Style:** Subtle arrows to show relationships without clutter
- **Collapsed Nodes:** Collapse less important nodes (like generic tags)
- **Layout:** Force-directed with some clustering for readability

## Tagging System 🏷️

### Project Tags
- `#project` - GitHub projects cloned from repositories
- `#in-progress` - Currently being worked on
- `#completed` - Finished projects
- `#planning` - Ideas not yet started

### Tool Tags
- `#github` - GitHub-related notes and repos
- `#python`, `#javascript`, etc. - Language-specific tags
- `#api`, `#database`, `#frontend` - Architecture layers

### Resource Tags
- `#docs` - Documentation pages
- `#tutorial` - Learning materials
- `#reference` - Quick lookup information

## Automation Notes ⚙️

When creating new projects, remember to:

1. Fill in the metadata (GitHub repo, tech stack, goals)
2. Add any relevant tags for graph organization
3. Link to related projects in your Profile note
4. Include any initial implementation notes or diagrams

## Maintenance Tips

- Run `#projects` tag filter weekly to see what needs attention
- Archive completed projects with a clear status update
- Keep the graph clean by removing outdated links
